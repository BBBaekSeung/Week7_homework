# Week 7 Retrospective

Picture Diary 프로젝트를 구현하며

agents 폴더 안에서 scene, image, video 역할을 나누고 pipeline.py에서 전체 흐름을 연결하는 구조로 장면 추출 → 이미지 생성 → 영상 생성 순서로 동작하는 파이프라인 구조를 구현함. API 키는 .env 파일로 관리하고 outputs 폴더에 결과물을 저장하는 방식도 직접 적용해봤다.


### 1. 파이프라인 구조

각 기능을 따로 구현 후 하나의 흐름으로 연결하는 게 중요!

* extract_scenes()
* generate_image()
* generate_video()
* picture_diary_pipeline()


### 2. 모듈 분리

* scene.py : 장면 추출
* image.py : 이미지 생성
* video.py : 영상 생성
* pipeline.py : 전체 실행


### 3. 환경 변수 관리

API 키를 코드에 직접 작성하지 않고 .env 파일을 통해 관리함.

.gitignore를 사용하면 주요정보가 GitHub에 업로드되지 않음!!


## 어려웠던 점 및 느낀 점
프로젝트 전체 구조나 파이프라인 흐름 자체는 가이드에 제시되어 있어서 이해하면 구현은 수월했지만 각 함수를 실제로 구현하는 게 오래걸렸다.
특히 이미지 생성 함수에서 OpenAI API를 호출하고 응답을 받아 저장하는 과정이 가장 헷갈렸다.
API 응답 구조를 확인하면서 원하는 데이터만 가져오는 부분도 여러 번 수정하였고, 함수마다 입력값과 반환값 형식을 맞추는 과정에서 오류가 발생해 여러번 확인이 필요했다. 
처음에는 각 기능을 구현하는 데만 집중했는데, 작업을 진행할수록 하나의 함수가 다른 함수와 어떻게 연결되는지를 신경썼다.
프로젝트 규모가 크지는 않았지만 파일 구조를 나누고 모듈별로 관리하는 방법을 경험해 볼 수 있었고, 앞으로 더 규모가 큰 프로젝트를 진행 할 때 파이프라인을 어떻게 설계할지에 대해 고민할 수 있는 시간이었다. 강의에서 배운 내용을 직접 적용하면서 이전에 알던 것들은 복습하고 파이프라인 구조와 API 활용 방식에 대해 이전보다 훨씬 익숙해질 수 있었던 시간이었다.
